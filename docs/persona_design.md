## 1. 设计理念

### 1.1 为什么需要人设？

EchoClass 的核心价值在于**还原真实课堂**。真实课堂中，每个学生都有独特的认知水平、性格特征和行为模式。如果所有虚拟学生都像"标准答案生成器"，师范生无法锻炼到关键的教学应变能力。

人设是 StudentAgent 的"灵魂"——它决定了虚拟学生会不会走神、会不会答错、会不会提出跑题的问题。**好的人设 = 真实的错误 + 可信的行为**。

### 1.2 为什么选这 6 种类型？

我们从两个维度交叉出典型学生画像：

| | 学业水平高 | 学业水平中 | 学业水平低 |
|---|---|---|---|
| **高互动** | ✅ 学霸型 | ✅ 活跃型 / 跑偏型 | — |
| **低互动** | ✅ 内向型 | — | ✅ 薄弱型 |
| **注意力差** | — | ✅ 走神型 | — |

6 个类型覆盖了师范生最常遇到的学生画像组合，确保每次模拟课堂都能练习到不同的教学策略：

| 人设 | 练习的教学能力 |
|---|---|
| **学霸型** | 如何利用优秀学生带动课堂？如何避免被单个学生"垄断"发言？ |
| **薄弱型** | 如何给予正向反馈？如何搭建认知脚手架？ |
| **走神型** | 如何用趣味引导拉回注意力？何时该干预、何时该放手？ |
| **活跃型** | 如何管理抢答行为？如何引导粗心学生检查过程？ |
| **内向型** | 如何创造安全的发言环境？如何识别"沉默≠不会"？ |
| **跑偏型** | 如何将跑题内容引回主线？如何保护创造性思维？ |

## 2. 皮亚杰认知发展理论的应用

### 2.1 理论基础

让·皮亚杰（Jean Piaget）将儿童认知发展分为四个阶段：

| 阶段 | 年龄范围 | 核心特征 |
|---|---|---|
| 感知运动阶段 | 0-2 岁 | 通过感官和动作认识世界 |
| 前运算阶段 | 2-7 岁 | 能使用符号，但逻辑不成熟 |
| **具体运算阶段** | **7-11 岁** | **能进行逻辑思维，但需要具体事物支持** |
| **形式运算阶段** | **11 岁以上** | **能进行抽象思维和假设推理** |

EchoClass 面向小学（7-12 岁）和初中（12-15 岁），正好跨越**具体运算阶段**和**形式运算阶段**的过渡期。

### 2.2 在人设中的具体体现

#### 具体运算阶段（小学生：P3-P4）

| 皮亚杰理论特征 | 人设中的行为映射 |
|---|---|
| **需要具体事物辅助思维** | 薄弱型学生套错公式 → 未能将抽象公式与具体情境关联 |
| **可逆性思维初步形成** | 学霸型能验算 → 具备了运算的可逆性 |
| **守恒概念建立中** | 走神型混淆周长和面积 → 守恒概念未完全稳固 |
| **分类和序列化能力** | 学霸型能有条理地组织解题步骤 |
| **自我中心逐渐消退** | 学霸型开始能考虑其他同学的不同解法（但还不充分） |

**设计要点**：小学人设的迷思概念集中在**具体操作层面**——算错、套错公式、读错题目。他们的错误是"做不到"而非"想不通"。

#### 形式运算阶段（初中生：J1-J2）

| 皮亚杰理论特征 | 人设中的行为映射 |
|---|---|
| **抽象思维能力** | 内向型能理解抽象概念但表述简略 |
| **假设-演绎推理** | 活跃型凭直觉判断而不验证 → 形式运算初期的不成熟 |
| **组合思维** | 跑偏型的过度类比 → 组合思维过于发散 |
| **命题思维** | 内向型能进行完整的逻辑证明（虽然跳步） |
| **元认知萌芽** | 活跃型答错后会自我反思"啊怎么会错" |

**设计要点**：初中人设的迷思概念集中在**抽象推理层面**——符号操作错误、过度泛化、逻辑跳步。他们的错误是"想歪了"而非"不会做"。

### 2.3 阶段过渡的设计考量

需要注意，皮亚杰的年龄划分是**近似值**，个体差异很大。在我们的设计中：

- **P3 学霸**（陈思远）表现出超越同龄人的逻辑能力，但仍受限于具体运算——他的过度自信来源于"已有方法的过度泛化"，而非真正的抽象推理
- **J1 活跃型**（张雨欣）处于形式运算的**初期**——能进行一定的抽象思维，但推理不严谨、容易凭直觉

## 3. 关键字段设计理由

### 3.1 核心身份字段

| 字段 | 设计理由 |
|---|---|
| `id` | UUID 唯一标识，便于 Agent 系统引用和持久化 |
| `name` | 中文姓名增加真实感，避免代号化的"学生1、学生2" |
| `gender` | 影响语言风格和人际互动模式 |
| `grade` | 决定知识范围和认知阶段的基准 |
| `age` | 配合 grade 提供更精确的发展水平参考 |

### 3.2 认知与学业字段

| 字段 | 设计理由 |
|---|---|
| `subject_level` | Agent 据此决定回答质量上限——"优秀"可以给出正确但可能跳步的回答，"薄弱"则大概率给出带有迷思概念的错误回答 |
| `cognitive_stage` | 皮亚杰阶段直接约束 Agent 的思维方式——具体运算阶段的学生不应产出抽象假设性推理 |
| `misconception_tendencies` | 最关键的教育学字段。每个迷思概念对应真实教学研究中发现的学生典型错误，让 Agent 犯"真实的错"而非随机错 |

### 3.3 行为与交互字段

| 字段 | 设计理由 |
|---|---|
| `attention_span` | Director Agent 据此决定该学生走神事件的触发频率 |
| `interaction_frequency` | 影响学生的主动发言概率和举手频率 |
| `behavior_traits` | 行为标签数组，Director 用于触发各种课堂事件（如"小动作多"→ 触发 distraction 事件） |
| `speech_style` | 直接注入 Prompt，让 LLM 输出符合人设的语言风格 |
| `catchphrases` | 口头禅让角色更鲜活，也便于前端做特征化展示 |

### 3.4 心理与背景字段

| 字段 | 设计理由 |
|---|---|
| `emotional_tendency` | 约束 Agent 输出的 emotion 字段分布，避免所有学生都"平静" |
| `learning_motivation` | 影响学生面对难题时的反应——内在驱动型会尝试探索，低动力型倾向放弃 |
| `personality` | 综合性格描述，作为 Prompt 的核心人设锚点 |
| `family_background` | 提供行为动机的深层解释，让 Agent 的角色扮演有"故事感" |

### 3.5 系统辅助字段

| 字段 | 设计理由 |
|---|---|
| `avatar_seed` | 前端头像生成的种子值，确保每个人设有固定且可复现的视觉形象 |
| `summary` | 一句话概括，用于 `/api/personas` 列表接口的快速展示 |

## 4. 迷思概念与人设的映射关系

迷思概念（Misconception）是学生头脑中与科学概念不一致的前概念。每个人设的 `misconception_tendencies` 基于以下原则设计：

### 4.1 映射规则

1. **与 subject_level 匹配**：
   - 优秀学生的迷思概念偏"高级"（如过度泛化、跳步遗漏）
   - 薄弱学生的迷思概念偏"基础"（如算法错误、概念混淆）

2. **与 cognitive_stage 匹配**：
   - 具体运算阶段的迷思概念围绕**操作性错误**（算错、套错公式）
   - 形式运算阶段的迷思概念围绕**推理性错误**（逻辑跳步、过度类比）

3. **与 grade 匹配**：
   - 迷思概念涉及的知识点不超出该年级的教学范围

### 4.2 后续扩展

W2 将建设独立的迷思概念库（`data/misconceptions/`），届时人设的 `misconception_tendencies` 将可以引用具体的迷思概念 ID，实现更精确的错误生成。
