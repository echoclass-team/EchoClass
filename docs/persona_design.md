## 1. 设计理念

### 1.1 为什么需要人设？

EchoClass 的核心价值在于**还原真实课堂**。真实课堂中，每个学生都有独特的认知水平、性格特征和行为模式。如果所有虚拟学生都像"标准答案生成器"，师范生无法锻炼到关键的教学应变能力。

人设是 StudentAgent 的"灵魂"——它决定了虚拟学生会不会走神、会不会答错、会不会提出跑题的问题。**好的人设 = 真实的错误 + 可信的行为**。

### 1.2 为什么选这 6 种类型？

我们从两个维度交叉出典型学生画像：

| | 学业水平高 | 学业水平中 | 学业水平低 |
|---|---|---|---|
| **高互动** | ✅ 学霸型 | ✅ 活跃型 / 跑偏型 | — |
| **低互动** | ✅ 内向型 | — | ✅ 薄弱型 |
| **注意力差** | — | ✅ 走神型 | — |

6 个类型覆盖了师范生最常遇到的学生画像组合，确保每次模拟课堂都能练习到不同的教学策略：

| 人设 | 1v1 答疑中练习的核心能力 |
|---|---|
| **学霸型** | 如何接住高阶追问（拓展、反例、类比）？如何避免被学生"带偏"到超出学段的话题？如何识别优秀学生看似懂了、实际跳步的隐性迷思？ |
| **薄弱型** | 如何识别学生问题背后的**错误前提**（`stuck_misconception`）？如何拆解到脚手架可接住的粒度？如何给出需要的正向反馈避免他提前放弃？ |
| **走神型** | 学生在答疑中中途跑题或注意力涣散时，如何用**生活化类比**拉回主线而不打击他？如何区分"真跑题"与"联想式推广"？ |
| **活跃型** | 面对凭直觉抢答、不验算的学生，如何追问其推理过程？如何让他"稳下来"反思而不打击积极性？ |
| **内向型** | 学生表达简略、跳步多时，如何用**启发式追问**诱出完整推理链？如何识别“沉默”背后是真懂还是不好意思追问？ |
| **跑偏型** | 学生提出 `off_topic` 问题时，如何在**尊重创造性 × 维护课堂主线**之间取平衡？如何把跑偏话题转化为教学资源？ |

## 2. 皮亚杰认知发展理论的应用

### 2.1 理论基础

让·皮亚杰（Jean Piaget）将儿童认知发展分为四个阶段：

| 阶段 | 年龄范围 | 核心特征 |
|---|---|---|
| 感知运动阶段 | 0-2 岁 | 通过感官和动作认识世界 |
| 前运算阶段 | 2-7 岁 | 能使用符号，但逻辑不成熟 |
| **具体运算阶段** | **7-11 岁** | **能进行逻辑思维，但需要具体事物支持** |
| **形式运算阶段** | **11 岁以上** | **能进行抽象思维和假设推理** |

EchoClass 面向小学（7-12 岁）和初中（12-15 岁），正好跨越**具体运算阶段**和**形式运算阶段**的过渡期。

### 2.2 在人设中的具体体现

#### 具体运算阶段（小学生：P3-P4）

| 皮亚杰理论特征 | 人设中的行为映射 |
|---|---|
| **需要具体事物辅助思维** | 薄弱型学生套错公式 → 未能将抽象公式与具体情境关联 |
| **可逆性思维初步形成** | 学霸型能验算 → 具备了运算的可逆性 |
| **守恒概念建立中** | 走神型混淆周长和面积 → 守恒概念未完全稳固 |
| **分类和序列化能力** | 学霸型能有条理地组织解题步骤 |
| **自我中心逐渐消退** | 学霸型开始能考虑其他同学的不同解法（但还不充分） |

**设计要点**：小学人设的迷思概念集中在**具体操作层面**——算错、套错公式、读错题目。他们的错误是"做不到"而非"想不通"。

#### 形式运算阶段（初中生：J1-J2）

| 皮亚杰理论特征 | 人设中的行为映射 |
|---|---|
| **抽象思维能力** | 内向型能理解抽象概念但表述简略 |
| **假设-演绎推理** | 活跃型凭直觉判断而不验证 → 形式运算初期的不成熟 |
| **组合思维** | 跑偏型的过度类比 → 组合思维过于发散 |
| **命题思维** | 内向型能进行完整的逻辑证明（虽然跳步） |
| **元认知萌芽** | 活跃型答错后会自我反思"啊怎么会错" |

**设计要点**：初中人设的迷思概念集中在**抽象推理层面**——符号操作错误、过度泛化、逻辑跳步。他们的错误是"想歪了"而非"不会做"。

### 2.3 阶段过渡的设计考量

需要注意，皮亚杰的年龄划分是**近似值**，个体差异很大。在我们的设计中：

- **P3 学霸**（陈思远）表现出超越同龄人的逻辑能力，但仍受限于具体运算——他的过度自信来源于"已有方法的过度泛化"，而非真正的抽象推理
- **J1 活跃型**（张雨欣）处于形式运算的**初期**——能进行一定的抽象思维，但推理不严谨、容易凭直觉

## 3. 关键字段设计理由

### 3.1 核心身份字段

| 字段 | 设计理由 |
|---|---|
| `id` | UUID 唯一标识，便于 Agent 系统引用和持久化 |
| `name` | 中文姓名增加真实感，避免代号化的"学生1、学生2" |
| `gender` | 影响语言风格和人际互动模式 |
| `grade` | 决定知识范围和认知阶段的基准 |
| `age` | 配合 grade 提供更精确的发展水平参考 |

### 3.2 认知与学业字段

| 字段 | 设计理由 |
|---|---|
| `subject_level` | Agent 据此决定回答质量上限——"优秀"可以给出正确但可能跳步的回答，"薄弱"则大概率给出带有迷思概念的错误回答 |
| `misconception_tendencies` | 最关键的教育学字段。每个迷思概念对应真实教学研究中发现的学生典型错误，让 Agent 犯"真实的错"而非随机错 |

> 注：原 18 字段 schema 中的 `cognitive_stage` 在 v1.1（2026-04-25）被移除。认知阶段现由 `stage_id` 关联的 `StageProfile.piaget_stage` 统一约束（学段共性层），避免与个体人设层重复。

### 3.3 行为与交互字段

| 字段 | 设计理由 |
|---|---|
| `attention_span` | 在 1v1 答疑场景下作为提问与对话风格提示注入 prompt：`short` 会在 ask 阶段增加跳步 / 跑题趋势，会在 chat 多轮过程中偶尔出现注意力涣散表达；`long` 则表现为追问有耐心 / 不轻易跳出主线。（转型前由 Director 调度走神事件，在 1v1 中由 prompt 直接指导。） |
| `behavior_traits` | 行为标签数组，是 prompt 里**人设鲜活度的主要质感来源**（如"抢着举手但并不知道答案"、"在本子上画画"）。会被 `behavior_traits_text` 拼接后注入 student_ask / student_chat / student_check 三个模板，引导 LLM 输出符合该人设的表达。 |
| `speech_style` | 直接注入三个 prompt，让 LLM 输出符合人设的语言风格。**v1.3 起与 `behavior_traits` 共同承担人设质感主职责**（原由 `personality` 加 `catchphrases` 合作承担） |

### 3.4 人设质感的联合承载（v1.3）

v1.3（2026-05-07）移除 `personality` / `catchphrases` / `family_background` 三个字段。原因：

- **`personality`**：与 `theory_anchors` 推入的 operational_rules 产生语义重叠。Posner / Vygotsky 等理论卡已在 `student_chat.j2` 里以“**必须遵守的行为约束**”优先级插入，再叠加一句“性格：XX”反而冲淡理论驱动。
- **`catchphrases`**：口头禅是 `speech_style` 的附属属性，拆为独立字段后以列表形式拼接到 prompt，LLM 容易机械式重复该口头禅（M2 dogfood 响应中出现 “哦哦哦这个是什么意思告诉我告诉我告诉我” 类伪鲜活）。现交由 `speech_style` 语义描述让 LLM 自然演绀。
- **`family_background`**：v1.1 起已为 deprecated，prompt 从未消费，作为可选字段存在反而增加人工设计负担。

> 迁移路径：原人设 JSON 里 性格 / 口头禅 的语义不会空白——`speech_style` 中已包含语调 / 口吻信息，`behavior_traits` 提供课堂动作标签，`theory_anchors` 驱动深层行为模式，`summary` 给出一句话高阶概括。四者联合承载之前 `personality` 一个字段试图一口气告诉 LLM 的信息。

> 历史变更：原 schema 中的 `emotional_tendency` / `learning_motivation` 在 v1.1（2026-04-25）被移除——三个 prompt 模板均未引用，且 1v1 对话输出 `DialogReplyResult` 不再携带 emotion 结构字段。

### 3.5 系统辅助字段

| 字段 | 设计理由 |
|---|---|
| `avatar_seed` | 前端头像生成的种子值，确保每个人设有固定且可复现的视觉形象 |
| `summary` | 一句话概括，用于 `/api/personas` 列表接口的快速展示 |

## 4. 迷思概念与人设的映射关系

迷思概念（Misconception）是学生头脑中与科学概念不一致的前概念。每个人设的 `misconception_tendencies` 基于以下原则设计：

### 4.1 映射规则

1. **与 subject_level 匹配**：
   - 优秀学生的迷思概念偏"高级"（如过度泛化、跳步遗漏）
   - 薄弱学生的迷思概念偏"基础"（如算法错误、概念混淆）

2. **与学段认知阶段匹配（由 stage_id 关联 StageProfile.piaget_stage）**：
   - 具体运算阶段的迷思概念围绕**操作性错误**（算错、套错公式）
   - 形式运算阶段的迷思概念围绕**推理性错误**（逻辑跳步、过度类比）

3. **与 grade 匹配**：
   - 迷思概念涉及的知识点不超出该年级的教学范围

### 4.2 与独立迷思库的关系

独立的学科 × 学段迷思概念库（`data/misconceptions/`，M1 已交付 21 份）是人设 `misconception_tendencies` 字段的补充与升级：

- **人设 `misconception_tendencies`**（字符串数组）：主观表述，表达该学生**偏好**哪类错误，仅被 `student_ask.j2` 注入 prompt
- **独立迷思库**（`Misconception` 结构化条目）：指向具体错误与干预路径，由 `match_misconceptions(subject, stage_id, key_points, ...)` 在每次生成问题时动态匹配最多 3 条，含 `id / typical_error / intervention_hint` 供学生问题与老师答疑脚手架使用

两者互补：迷思库负责标出**该课客观上需避踩的雷**，人设 `misconception_tendencies` 负责刻画**该学生个体主观上更偏向哪类雷**。
