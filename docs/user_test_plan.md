## 0. TL;DR

- **目的**：在 M3 上线前，验证 EchoClass 的 1v1 答疑陪练闭环对**真实师范生**是否可用、是否有价值、是否会被推荐给同学
- **对象**：3–5 名在校师范生（数学 / 物理 / 化学 / 生物 / 历史 / 语文 / 英语 / 政治 任意学科），覆盖**至少 2 个学段**的实际授课目标
- **形式**：1 对 1 远程 + 屏幕共享 + 录屏，每场 **45 分钟**
- **度量**：SUS（系统可用性量表）+ NPS（推荐意愿）+ 半结构化访谈 + 任务完成率
- **产出**：每场一份反馈记录卡 + 汇总到 `docs/user_test_report.md`（#136 交付）
- **验收线**（本 Issue #135）：方案 PR approve + 至少 **3 人** 预约确认（招募名单见 §6）

## 1. 测试目标与假设

### 1.1 我们想验证的核心问题

按重要度排序：

1. **价值假设**：师范生是否认为"虚拟学生答疑"比"自己空想 / 同学互测"更能暴露备课盲区？
2. **可用假设**：从登录到看到评估报告，**新用户能否在 10 分钟内独立完成一次完整闭环**，无需口头指导？
3. **真实感假设**：学生 Agent 的提问 / 反问 / 跑题 / `[懂了]` 信号，是否被师范生认为"像我教过的某个真学生"？
4. **评估有效性假设**：EvaluatorAgent 给出的 5 维评分（MR/KC/RR/TQ/SS，详见 [`rubric_v0.md`](./rubric_v0.md)）是否被师范生认为"有道理 / 看得懂 / 愿意据此改进"？
5. **推荐意愿**：参与者愿意把 EchoClass 推荐给同学 / 学弟学妹（NPS ≥ 7）的比例

### 1.2 不在本轮测试范围内

为聚焦核心闭环，**本轮不验证**：

- 多人协作 / 教研组协同（M4+）
- 移动端 / 平板适配（M3 仅桌面 Web）
- 语音 / TTS 输入（roadmap）
- 课堂直播场景（roadmap）

### 1.3 风险提示

| 风险 | 缓解措施 |
|---|---|
| 招募 3 人不足 | §6 双渠道并行 + 50 元红包激励，最低门槛 2 人也跑 |
| 用户上传的教案学科 / 学段不在 18 persona × 15 lesson 覆盖范围 | §3.2 任务 1 提供 fallback：可使用预置教案 |
| 测试中后端 / WS 挂掉 | §4 主持人手册含降级流程：切到本地 mock + 留作补救场次 |
| 评估接口尚未真实落地（依赖 #124/#125） | 见 §7 排期：#136 执行需在 #124+#125 合入后；本 Issue 仅交付方案 |

## 2. 招募标准

### 2.1 主标准（必须满足）

| 维度 | 要求 |
|---|---|
| **身份** | 在读师范生（本科 / 硕士均可） |
| **年级** | 大三 / 大四 / 研一 / 研二（已上过教学法课，正在或即将进入实习） |
| **学科** | 至少能教 [小学 / 初中 / 高中] **任意一个学段**的 [数 / 理 / 化 / 生 / 语 / 英 / 史 / 政] 任意一科 |
| **数量** | 3–5 人 |
| **设备** | 笔记本 / 台式机 + 网络，能进行 45 分钟屏幕共享 |
| **时间** | 一次连续 45 分钟，可错峰 |

### 2.2 多样性配额（建议而非硬性）

为暴露不同盲点，5 人配额建议覆盖：

- **学段**：至少含 1 名小学、1 名中学（初/高均可）
- **学科**：至少 2 个学科
- **教学经验**：≥ 1 名"已实习过"+ ≥ 1 名"未实习"
- **性别**：尽量平衡（不强制）

### 2.3 排除标准

- 已参与过 M2 内部 dogfood 的同学（避免熟悉度偏差）
- 团队成员的本组同学（避免人情分）
- 完全没修过教学法 / 学科教学论的同学（缺少基本备课语境）

## 3. 任务脚本（45 分钟标准流程）

> 主持人 = 团队成员（C 或 B）；参与者 = 师范生。
> 全程屏幕共享 + 经授权后录屏；主持人**不主动指导操作**，仅在卡死 ≥ 60s 时提示。

### 3.1 时间分配

| 段落 | 时长 | 内容 |
|---|---|---|
| §3.2 暖场 + 知情同意 | 5 min | 介绍背景、签知情同意（口头确认即可） |
| §3.3 任务 1：登录 + 上传教案 | 8 min | 登录账号 → 上传一份**自己**或预置教案 |
| §3.4 任务 2：创建陪练 session | 5 min | 选学生人设 → 开始陪练 |
| §3.5 任务 3：1v1 答疑（≥ 2 个学生） | 15 min | 与至少 2 个虚拟学生完成答疑 |
| §3.6 任务 4：查看评估报告 | 5 min | 进入复盘页，理解 5 维评分 |
| §3.7 SUS + NPS 量表 | 4 min | 在线表单填写 |
| §3.8 半结构化访谈 | 3 min | §5.3 大纲，重点追问"为什么" |

> 总计 45 分钟。**主持人务必在 §3.5 控时**——这一段最容易超时。

### 3.2 暖场 + 知情同意（5 min）

主持人脚本（**只念一次**，不要发挥）：

> "你好，感谢你抽时间帮我们测试 EchoClass。我们做了一个 AI 答疑陪练系统：你上传一份教案，系统会模拟一群有典型困惑的虚拟学生跟你对答疑，结束后给你一份诊断报告。
> 接下来 45 分钟，我会请你完成 4 个任务，全程屏幕共享 + 录屏，**录屏只用于团队复盘，不会公开**。任务里有不顺手的地方，**请你大声吐槽**，越尖锐越好——我们要的就是这个。
> 我不会教你怎么操作；如果你卡住了我会提示，但请先尝试自己摸索。准备好了我们开始？"

知情同意 checklist（口头确认即可）：

- [ ] 同意录屏 + 录音
- [ ] 同意将匿名化反馈写入 `docs/user_test_report.md`
- [ ] 知道可随时叫停
- [ ] 收到 50 元感谢红包（测试结束后发放）

### 3.3 任务 1：登录 + 上传教案（8 min）

**指令（原话给参与者）**：

> "请用我提前发给你的账号登录 EchoClass，然后上传一份你**自己写过的或正在备的**教案。如果手边没有，桌面上有 3 份预置教案可以用。"

**预期路径**（基于 [`api_contract.md`](./api_contract.md) §2 + [`handoff_b_m2.md`](./handoff_b_m2.md) §4.1）：

1. 浏览器打开 `https://echoclass-test.example.com/`
2. 登录页输入账号密码（详见 §6 测试账号）
3. 跳转至 `/setup`，看到上传区
4. 拖拽 / 点选 PDF / Markdown / TXT
5. 等待 5–15s LLM 抽取 LessonMeta
6. 看到解析卡片：标题 / 学科 / 学段 / 关键知识点 / 重难点

**观察点**：

- 登录页是否一眼能找到登录按钮？
- 上传区视觉是否引导明确？
- 等待解析时是否焦虑（loading 提示是否够）？
- 看到解析结果是否有**惊喜或质疑**（关键知识点抽取是否符合预期）？

**fallback**：参与者教案被解析为奇怪结果 → 主持人引导改用预置教案。

### 3.4 任务 2：创建陪练 session（5 min）

**指令**：

> "系统会推荐 3 个虚拟学生。你看一下推荐的合不合适，可以**改学段、加学生、减学生**，然后开始陪练。"

**预期路径**：`/setup/personas` → 看到 3 张学生卡（含学段 / 学科水平 / 头像 / persona 标签）→ 调整 → 点「开始陪练」→ `/qa/{session_id}`。

**观察点**：

- 推荐学生的人设标签（如 `restless` / `xueba` / `anxious`）是否能让师范生**一眼判断"这是个什么样的孩子"**？
- 调整功能是否被注意到？
- 「开始陪练」按钮位置是否被找到？

### 3.5 任务 3：1v1 答疑（15 min，**核心**）

**指令**：

> "现在你看到 N 个学生在等你答疑（红点表示问题数）。请挑**至少 2 个不同的学生**，跟他们各完成 1–2 轮答疑。学生说 `[懂了]` 表示这个问题解决了。如果你觉得某个问题没法继续，可以放弃。"

**预期路径**（详见 [`handoff_b_m2.md`](./handoff_b_m2.md) §5）：

1. 左侧学生列表，红点显示待解答数量
2. 点学生 → 中间区域显示该学生第一个问题
3. 输入框打字回复 → 学生流式回复（打字机效果）
4. 多轮往返，学生可能继续追问 / 跑题 / 表达困惑
5. 学生输出 `[懂了]` → 弹 toast 提示自解决
6. 切换到第 2 个学生，原对话保留
7. 主持人控时 15 min

**观察点**（**这是本轮测试最重的段落**）：

| 维度 | 关注问题 |
|---|---|
| 真实感 | 学生回复"像不像真学生"？哪一句最像 / 最不像？ |
| 流式体验 | 打字机速度是否舒服？`[懂了]` 视觉处理是否清楚？ |
| 共情压力 | 师范生在被反问 / 跑题时是否感到压力（**这是产品价值的关键**）？ |
| 多对话切换 | 切回原对话进度是否丢？心智负担多大？ |
| 错误恢复 | 学生回复出现明显幻觉 / 跑偏时，师范生如何处理？是否有"重生"按钮？ |

**主持人禁忌**：

- ❌ 不要解释"系统设计"
- ❌ 不要替师范生想"学生为什么这么问"
- ✅ 师范生卡 ≥ 60s 才提示："你可以试试输入「我换个角度问」"

### 3.6 任务 4：查看评估报告（5 min）

**指令**：

> "现在退出 session，看看系统对你这次答疑的评估。请你边看边告诉我：哪一项你觉得**有道理**，哪一项你觉得**离谱**，哪一项你**看不懂**。"

**预期路径**：`/qa/{session_id}/summary` → 5 维评分卡（MR / KC / RR / TQ / SS，详见 [`rubric_v0.md`](./rubric_v0.md)）+ 每维证据片段（excerpt）+ 自由文本反馈（FeedbackAgent 输出）。

**观察点**：

- 5 维度名字（迷思破除 / 重点覆盖 / 解决率 / 师范生提问质量 / 学生满意度）是否一眼懂？
- 0–4 分的等级描述是否被认可？
- 每维的「证据片段」是否真的回溯到了**关键对话点**？
- FeedbackAgent 的自由文本反馈是否**具体可操作**（vs. 套话）？
- 师范生看完后**是否产生改进意愿**（→ 这是产品 stickiness 的关键）？

### 3.7 SUS + NPS 量表（4 min）

主持人发链接（见 §5.1 / §5.2），参与者在线填写。主持人**全程不评论**。

### 3.8 半结构化访谈（3 min）

§5.3 大纲，**只问 3 个核心问题**，超时立刻收尾。

## 5. 度量工具

### 5.1 SUS（系统可用性量表，10 题中文版）

> 来源：Brooke (1996) System Usability Scale，中文翻译参考 Sauro & Lewis (2016)。
> **使用规则**：5 级 Likert（1=非常不同意，5=非常同意）；奇数题正向计分（得分−1），偶数题反向计分（5−得分）；总和 × 2.5 得 0–100 分。
> **基线**：68 = 行业平均；80+ 优秀；50- 红线。

| # | 题目 | 极性 |
|---|---|---|
| 1 | 我愿意经常使用这个系统 | 正 |
| 2 | 我觉得这个系统过于复杂 | 反 |
| 3 | 我觉得这个系统使用起来很容易 | 正 |
| 4 | 我需要技术支持人员的帮助才能使用这个系统 | 反 |
| 5 | 我觉得系统中的各项功能整合得很好 | 正 |
| 6 | 我觉得系统中存在太多前后不一致的地方 | 反 |
| 7 | 我认为大多数人能很快学会使用这个系统 | 正 |
| 8 | 我觉得这个系统使用起来非常笨拙 | 反 |
| 9 | 我使用这个系统时充满信心 | 正 |
| 10 | 我需要先学很多东西才能上手这个系统 | 反 |

> 在线表单（待开通后填入）：`https://docs.qq.com/form/page/__SUS_FORM_ID__`

### 5.2 NPS（推荐意愿，单题）

> "假设有一位同年级 / 同学科的师范生学弟学妹问你 EchoClass 值不值得用，你**0–10 分**有多大可能推荐他用？"
>
> - 9–10 分：推荐者
> - 7–8 分：被动者
> - 0–6 分：贬损者
>
> NPS = 推荐者占比 − 贬损者占比。**M3 验收线：NPS ≥ 0**（即推荐者 ≥ 贬损者）。

紧跟一个开放追问：**"差几分能让你给到 10？"** ——这一句是金矿，**主持人务必逐字记下**。

### 5.3 半结构化访谈（3 个核心问题）

按顺序问，**不要跳题**：

1. **价值**："如果以后你备课，你会**真的**用 EchoClass 吗？什么场景下会用 / 不会用？"
2. **痛点**："今天 45 分钟里，**最让你想砸键盘**的瞬间是哪一个？"
3. **改进**："如果只让我们改一个东西就能让你推荐分 +2，你会让我们改什么？"

### 5.4 任务完成率（主持人客观记录）

| 任务 | 通过标准 | P1 | P2 | P3 | P4 | P5 |
|---|---|---|---|---|---|---|
| T1 登录 + 上传 | 2 分钟内进入解析卡片 |  |  |  |  |  |
| T2 创建 session | 1 分钟内进入 `/qa/{id}` |  |  |  |  |  |
| T3 完成 ≥ 2 学生答疑 | ≥ 2 个学生有 ≥ 1 轮往返 |  |  |  |  |  |
| T4 看懂评估报告 | 能口头复述 5 维含义 ≥ 3 个 |  |  |  |  |  |

## 6. 招募计划

### 6.1 渠道

| 渠道 | 估算可触达 | 转化预期 | 负责人 |
