# EchoClass

> AI-powered 1v1 Q&A coach for pre-service teachers.  
> 师范生 1v1 答疑陪练系统 —— 让未来的教师在走上讲台前，先反复练习"被学生追问"。

## 项目简介

EchoClass 是一个面向师范生的 AI 答疑陪练系统。它不是让 AI 直接替老师讲课，而是让 AI 扮演不同学段、不同性格、不同知识基础的"虚拟学生"，围绕师范生上传的教案主动提问，并在 1v1 对话中持续追问、暴露迷思、等待老师用合适的脚手架解释清楚。

系统的目标是帮助师范生在真正走上讲台前，反复练习以下能力：

- **识别学生迷思**：从学生看似简单的问题中判断其错误前提。
- **分级解释概念**：根据学段认知水平调整语言、例子和抽象程度。
- **应对连续追问**：在多轮答疑中保持教学目标、情绪支持和解释质量。
- **复盘教学表现**：通过评估报告与反馈建议发现自己的答疑盲区。

## 核心痛点

师范生在校内能学习大量教育学理论，也能进行试讲训练，但在真正实习前，很少有机会面对"会问真实问题的学生"进行反复答疑。

EchoClass 关注的是比试讲更细、更难练的场景：**学生不懂、追问、误解、半懂不懂时，老师如何把概念讲到点子上。**

主要痛点包括：

- **同伴扮演存在温室效应**  
  师范生互相扮演"小学生"时，知识背景太接近，往往提不出真实儿童式困惑。

- **答疑场景比试讲更稀缺**  
  很多师范生经历过试讲，却很少经历完整的"被学生连续追问"。

- **迷思诊断能力难以靠背理论获得**  
  "识别错误前提 → 制造认知冲突 → 给出反例或类比 → 引导学生复述"这套能力，需要大量对话实践。

- **缺少可复盘的训练闭环**  
  传统训练结束后常停留在主观评价，缺少结构化证据、评分维度和可执行改进建议。

## 使用流程

```text
[1] 上传教案（PDF / Markdown / TXT）
    → 自动抽取学科、学段、教学目标、重点、难点

[2] 选择虚拟学生
    → 基于学段认知特征与学生人设生成不同类型的学习者

[3] 学生主动提问
    → 系统根据教案和迷思库生成候选问题

[4] 进入 1v1 答疑对话
    → 老师回答，学生根据理解程度继续追问或表达困惑

[5] 学生真正理解后结束该题
    → 学生以自然语言表示理解，系统记录解决方式

[6] 结束 session 后进入复盘
    → 查看对话回放、评估报告、反馈建议，并可导出 Markdown
```

## 关键设计

### 1. 双层虚拟学生建模

EchoClass 使用"学段共性 × 个体差异"的双层人设建模方式。

- **学段共性**：根据皮亚杰认知发展阶段、语言表达水平和常见思维方式，为不同学段设置认知边界。
- **个体差异**：为学生叠加性格、口头禅、学习信心、迷思倾向、情绪反应等特征。

这使得虚拟学生既不会"超模"到小学生讲高等数学，也不会变成机械问答机器人。

### 2. 学科迷思库驱动的问题生成

系统维护学科 × 学段的迷思概念库，用于生成更接近真实课堂的错误前提与困惑表达。

例如，学生不会只问"这题怎么做"，而可能表现为：

- 把分母大小误认为分数大小。
- 把速度和力混为一谈。
- 把函数图像的局部趋势误认为整体性质。

这些迷思会被注入学生提问与对话行为中，逼迫师范生练习真正的概念诊断。

### 3. 宽生成、自检与多样性筛选

学生问题不是一次性生成后直接使用，而是经过"宽生成 → 自检 → 多样性筛选"的流程：

1. 生成多个候选问题。
2. 让模型自评其人设贴合度、教育价值和教案相关性。
3. 过滤低质量问题。
4. 按问题类别与难度做多样性选择。

这样可以避免问题过于重复，也能让答疑训练覆盖更多教学场景。

### 4. 面向脚手架能力的对话约束

对于包含错误前提的问题，学生不会因为老师简单给出答案就立刻"懂了"。系统会要求学生在错误前提没有被指出、缺少反例或类比、没有完成复述前继续表现困惑。

这使训练重点从"把正确答案说出来"转向"让学生真的理解"。

### 5. 复盘评估与反馈建议

一次答疑结束后，EchoClass 会生成结构化复盘内容，包括：

- 对话历史回放。
- 学生问题解决状态。
- Rubric 维度评分。
- 具体证据片段。
- 师范生反馈建议。
- Markdown 导出材料。

评估不追求给出单一总分，而是关注多个教学维度，例如迷思破除、重点覆盖、解决率、提问质量和学生满意度。

### 6. 教育学理论锚点

系统设计参考了多种教育学理论，并将其用于人设、迷思和评估设计中：

- Piaget 认知发展阶段。
- Vygotsky 最近发展区与脚手架。
- Posner 概念变化模型。
- Chi 迷思分类。
- Bandura 自我效能。
- Weiner 归因理论。
- Pekrun 学业情绪。
- Deci-Ryan 自我决定理论。

这些理论不是作为装饰性概念出现，而是用于约束虚拟学生的认知边界、情绪反应、错误类型和复盘评价标准。

## 功能概览

- **教案解析**：支持 PDF / Markdown / TXT 教案上传与结构化抽取。
- **虚拟学生选择**：提供跨学段、跨性格、跨学习水平的学生人设。
- **主动提问生成**：根据教案重点、难点和迷思库生成学生问题。
- **1v1 流式答疑**：通过 WebSocket 实现实时对话体验。
- **多学生 session**：支持在多个学生对话之间切换和管理进度。
- **自动评估**：结束 session 后自动触发评估与反馈生成。
- **复盘页面**：展示历史 session、对话回放、评分与建议。
- **Markdown 导出**：将对话、评估和反馈整理成可保存的复盘文档。
- **Demo 种子数据**：内置 good / mid / bad 三档样本，便于演示与测试。

## 项目定位

EchoClass 的核心不是"AI 替老师讲"，而是"AI 作为可反复练习的学生"，帮助师范生在低风险环境里经历更多真实答疑情境。

它希望把师范生训练从一次性的试讲，推进到可重复、可量化、可复盘的教学能力练习：

> 让未来的教师，在第一次面对真实学生之前，已经被虚拟学生认真追问过很多次。
